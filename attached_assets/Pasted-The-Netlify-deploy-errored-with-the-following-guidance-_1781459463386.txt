Diagnosis
- Primary failure: npm could not download packages because it attempted to resolve a private/internal registry host package-firewall.replit.local which is not reachable from Netlify. The network/DNS error is the fatal error that stops install ([lines 251–256](#L251-L256), [line 254](#L254)).
- Secondary problem (warning that should be addressed): your build used Node v18 but react-router / react-router-dom state they require Node >=20 ([lines 12](#L12), [240–249](#L240-L249)). This is a compatibility warning that can become a hard failure depending on package behaviour.

Cause
- A registry override (likely an .npmrc or lockfile config created by Replit or similar) points npm to package-firewall.replit.local which only exists inside Replit. Netlify cannot resolve that host and npm install fails.
- Node version on Netlify is 18.17.1 but some packages request Node >=20.

Solution (steps to fix)
1) Verify repository does not force the Replit registry
   - Check for an .npmrc in the repo root that sets a custom registry:
     - Verify file exists: https://github.com/tonymicah81-ui/Demax02/blob/main/.npmrc
     - If present, remove it or change the registry to the public npm registry and commit the change.
   - If .npmrc is not present, check package-lock.json (or yarn.lock/pnpm-lock.yaml) for occurrences of package-firewall.replit.local and regenerate the lockfile after setting the correct registry.

2) Replace or remove the Replit registry entry
   - To set the registry to the public npm registry, create/edit .npmrc in repo root with:
```text
registry=https://registry.npmjs.org/
```
   - Commit and push:
```bash
git add .npmrc
git commit -m "Use public npm registry for CI builds"
git push
```
   - Or remove the .npmrc entirely if it was injected by Replit:
```bash
git rm .npmrc
git commit -m "Remove Replit npm registry override"
git push
```
   - Alternatively, on Netlify you can set the registry at build time via a build command:
```bash
npm config set registry https://registry.npmjs.org/ && npm ci
```
   - If your lockfile references the Replit registry, regenerate it locally after ensuring the registry is set to npmjs.org:
```bash
# locally
npm config set registry https://registry.npmjs.org/
rm package-lock.json
npm install
git add package-lock.json
git commit -m "Regenerate lockfile using public npm registry"
git push
```

3) Fix the Node version mismatch (react-router requires Node >=20)
   - In Netlify you can set the Node version via netlify.toml or provide an .nvmrc. See Netlify docs: https://docs.netlify.com/configure-builds/manage-dependencies/#node-js-and-javascript
   - Example using netlify.toml (add to repo root or update existing):
```toml
[build.environment]
  NODE_VERSION = "20"
```
   - Or add an .nvmrc with the version:
```bash
echo "20" > .nvmrc
git add .nvmrc
git commit -m "Use Node 20 for Netlify builds"
git push
```
   - After this Netlify will use Node 20.x and the engine warnings should be resolved.

4) Verify packages present in package.json before changing registry (per checklist)
   - Confirm the packages you expect are in package.json (e.g. react-router, react-router-dom): https://github.com/tonymicah81-ui/Demax02/blob/main/package.json
   - If a package is missing, add it to package.json and commit before trying to change build configuration.

5) Re-run the Netlify build
   - After removing/adjusting .npmrc (or regenerating lockfile) and updating Node version config, trigger a new deploy on Netlify.
