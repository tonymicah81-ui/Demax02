The Netlify deploy errored, with the following guidance provided:

- Relevant log lines
  - [line 86–89](#L86-L89) shows esbuild failing to resolve "cloudinary".
  - [line 96–101](#L96-L101) is Netlify's explanation that a Function uses "cloudinary" but the dependency wasn't installed.
  - [line 108](#L108) repeats the same bundler error.

- Error type and cause
  - Type: missing dependency at function-bundling time (esbuild/Netlify Functions).
  - Cause: netlify/functions/sign-cloudinary.ts imports "cloudinary" but that package was not available to the functions bundler, so esbuild cannot resolve it. Netlify does not automatically install dependencies from a package.json inside a Functions folder, so either the package isn't declared in the reachable package.json or it wasn't installed for functions.

- Solutions (pick one)

  1) Recommended — install cloudinary at the project root (simplest)
     - Verify cloudinary is in package.json or that the functions package.json was committed:
       - Open your repo's top-level package.json and/or netlify/functions/package.json and confirm "cloudinary" appears in "dependencies".
       - Repo file links: package.json (root) and the function file: https://github.com/tonymicah81-ui/Demax02/blob/main/netlify/functions/sign-cloudinary.ts
     - If not present at root, install and commit:
```bash
# from repo root
npm install --save cloudinary
git add package.json package-lock.json
git commit -m "Add cloudinary runtime dependency for Netlify Function"
git push
```
     - Re-deploy. Netlify will bundle the function using the project dependencies and the error should go away.

  2) If you keep a separate package.json inside netlify/functions
     - Verify netlify/functions/package.json exists and lists "cloudinary".
     - Have Netlify install the function's dependencies during build. Either:
       - Add the Netlify plugin that installs functions dependencies by adding to netlify.toml:
```toml
[[plugins]]
package = "@netlify/plugin-functions-install-core"
```
         Then commit netlify.toml and deploy. The plugin will install dependencies declared in netlify/functions/package.json before bundling.
       - Or run an install step in your build command (in Netlify UI or netlify.toml). Example build command:
```toml
[build]
  command = "cd netlify/functions && npm install && cd ../.. && npm run build"
  publish = "dist"
```
         (Or add a root-level prebuild script that runs npm install inside the functions folder.)

  3) Alternative: mark "cloudinary" as external (only after verifying the package is present)
     - First verify the package is installed/declared (see step above).
     - If you prefer not to bundle cloudinary into the function, add it to functions external modules in netlify.toml:
```toml
[functions]
  external_node_modules = ["cloudinary"]
```
     - Commit and redeploy. This tells Netlify’s bundler not to bundle "cloudinary" (it will require it at runtime), so ensure the runtime environment has the dependency available.

Notes
- The quickest fix is to ensure "cloudinary" is listed in a package.json that Netlify will install (preferably the root package.json) and commit the updated package.json/package-lock.json, then redeploy.
- No Node version changes appear necessary from the logs. If you do need to change Node versions later, use Netlify docs: https://docs.netlify.com/configure-builds/manage-dependencies/#node-js-and-javascript
