The Netlify deploy errored, with the following guidance provided:

Diagnosis
- Netlify installed Node v22.22.3 for the build ([lines 12–16](#L12-L16)).
- npm failed during install with an internal error "Exit handler never called" and aborted the install ([lines 19–24](#L19-L24)). The build then failed at dependency installation ([lines 20, 26](#L20-L26)).
- Likely cause: an incompatibility/bug between the automatically selected Node/npm version (Node 22 + npm 10) and either npm itself or one of your dependencies. The repo does not pin a Node version, so Netlify picked Node 22 which can trigger npm internal errors for some setups.

Solution
1. Pin a stable LTS Node version for Netlify (recommended: 18.x or 20.x).
   - Option A — add a .nvmrc file at repo root (simplest):
     ```text
     18
     ```
     Commit and push.
   - Option B — add engines to package.json:
     ```json
     {
       "engines": {
         "node": ">=18 <21",
         "npm": ">=9"
       }
     }
     ```
     Commit and push.
   - Option C — set in netlify.toml:
     ```toml
     [build.environment]
     NODE_VERSION = "18"
     ```
     Commit and push.

2. Clear the Netlify build cache and trigger a new deploy.
   - In Netlify UI: Deploys → Trigger deploy → Clear cache and deploy site.
   - Or push a new commit after adding the Node pin.

3. Verify locally before redeploying:
   - Install Node 18 or 20 locally (nvm use 18).
   - Run:
     ```bash
     npm ci
     npm run build
     ```
   - Confirm install/build succeed.

4. If the error persists after pinning Node:
   - Inspect the full npm debug log referenced in the build log for details (/opt/buildhome/.npm/_logs/... was printed in the log) and share it.
   - Ensure your dependencies are correct in package.json. If a build-level external config is later required (e.g., rollup externals), first verify the package is listed in package.json or that the required file was committed to the repo before adding it to any external config.

Netlify docs for changing Node versions:
https://docs.netlify.com/configure-builds/manage-dependencies/#node-js-and-javascript

Why this will fix it
Pinning to Node 18/20 prevents Netlify from auto-using Node 22 + npm 10 where this npm internal error appears. Many build errors like "Exit handler never called" are resolved by using a known LTS Node + npm combination.
